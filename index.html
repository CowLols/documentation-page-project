Page Design

Pages use tables to act as lists and image folders.

Images that are added must open the image file in a new tab when clicked on.

Images must be in "figure" elements,"figcaption" elements must be used to describe the image aswell.

Links must have the "link" class.

Code demo below

class="link"

Pages must also have the correct CSS properties. This is to make sure pages are not different in design and to make sure they don't cause any confusion.


An example can be found below:

.history { width:100%; height:100%; border:1px solid; border-collapse:separate; padding:6px; }

The .history class is used for a timeline of events. This is how it is styled.


Image galleries use a grid style to present images. An example of this can be found below:

#gallery { display: grid; grid-template-columns: repeat(3,1fr); grid-auto-flow: dense; } .galleryimg { width: 100%; height: 100%; border: 2px solid white; z-index: 998 }
Adding Content

Content must be approved and reviewed before being added to the main site.

Any proposed changes will be sent for review.

Your changes must be:

  • Relevant to the topic.
  • Have correct information.
  • Contribute a significant detail or fix a problem with the article.

Edits must also adhere to the page design standards. This can be viewed Here.
